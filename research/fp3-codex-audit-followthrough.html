#5 — Harden the preview upload

crates/orchestrator/src/routes/preview.rs:

  • Magic bytes guard: reject the upload if the first 5 bytes are not %PDF-. Return 415 Unsupported Media Type with hint "Only standard PDF files are accepted (first bytes must be %PDF-).".
  • Low-text fallback: if pdf_to_text yields < 200 characters, return 422 Unprocessable Entity with hint "PDF appears empty or image-only; OCR is not supported in the preview path.".
  • Rate-limit / timeout mapping: map LLMError::RateLimited → 503 with a Retry-After header + hint; map LLMError::Timeout → 504. Both already differentiated in the adapter; just route them.

Playwright assertions fold into tests/e2e-web/upload.spec.ts (§2): non-PDF + oversize already there; add a "low-text PDF" test that uses a near-empty PDF and asserts the 422 hint text renders.

#6 — Publish E2E against a disposable test repo (opt-in)

scripts/publish-e2e.sh (new, opt-in via env):

  1. Require GITHUB_TOKEN, GROKRXIV_REVIEWS_OWNER, GROKRXIV_REVIEWS_REPO (a test repo).
  2. Run M1 against a real arxiv-id to produce awaiting_moderation.
  3. grokrxiv approve <review_id> → asserts a real PR URL is returned and reachable via gh pr view.
  4. gh pr merge --merge <pr_number>.
  5. Generate a synthetic GitHub pull_request.closed webhook payload, sign with HMAC matching GITHUB_WEBHOOK_SECRET, POST to localhost:8080/webhook/github.
  6. Assert select status, published_at from reviews where id=<r> → published, published_at not null.
  7. curl /api/v1/reviews/<r> → 200 with status: "published".
