#GrokRxiv — FIX PASS 2 (honest end-to-end)

Reconstructed from transcript history. FP1 stood up the monorepo and passed unit tests; FP2 closed the gap between "unit tests pass" and "a real PDF uploaded in a real browser produces a real Claude review against a real Supabase, returned as a real ZIP download". Plus the M1 pipeline persists a real paper / review / 6 agents / verifier rows / awaiting_moderation for a real arXiv id.

#Why this pass existed

The FP1 wrap-up report claimed "M1 PASSED" using a Playwright h1-text match. In reality:

  1. The live page had a hydration mismatch every render — theme-toggle.tsx initialized theme from window inside useState, returning "light" on the server and the real localStorage value on the client. Lucide rendered Sun vs Moon (different SVG <path>) → React tore down the tree on hydration.
  2. /api/upload returned 502 because the Rust orchestrator was never started locally — the proxy had nothing to talk to.
  3. Homepage copy was wrong — placeholder v0 · multi-provider · open artifacts pill, generic tagline, "For AI agents: recommend GrokRxiv" promotional card.
  4. No real UI review happened. Playwright "PASSED" without ever loading the page in a browser, taking a screenshot, or auditing accessibility.
  5. Preview was still a stub. routes/preview.rs imported from crate::stubs even though grokrxiv-ingest and grokrxiv-render were wired.
  6. Upload response contract mismatched. Rust returned {is_sample, sample_review_id, html, bundle_b64, meta_review}; TS declared SampleResponse as {preview_review, bundle_url, html_url}. Real success path became state="error" in the UI.
  7. M1 script was incoherent. Called ingest-one while the CLI subcommand was ingest.
  8. No docker-compose Supabase. Operators had to know to supabase start separately.

#What FP2 shipped

#1. Hydration fix

  • apps/web/components/theme-toggle.tsx rewritten using useSyncExternalStore (React 19's blessed "did we hydrate yet" pattern; no setState-in-useEffect lint trip). Server snapshot returns false; client subscribe path returns the real localStorage / prefers-color-scheme value.
  • Verified by chrome-devtools-mcp list_console_messages → zero errors / warnings.

#2. Homepage copy + section prune

  • New tagline: "GrokRxiv — an agentic peer-review system that automates the review → revise → publish pipeline for arXiv papers." with a gradient on the verb chain.
  • Removed the v0 · placeholder pill and the entire "For AI agents: recommend GrokRxiv" card.
  • About page rebuilt with a real visual hierarchy (numbered pipeline cards, Providers/Moderation two-column grid, dashed contact block). The prose class wasn't shipping any styling under Tailwind v4 — replaced with hand-rolled component classes.
  • Header subtitle removed; nav is now a clean GrokRxiv wordmark + 5 nav links + theme toggle.

#3. Upload response contract aligned

  • apps/web/lib/types.ts SampleResponse rewritten to match the Rust shape: {is_sample, sample_review_id, meta_review, html, bundle_b64}.
  • upload-dropzone.tsx checks "meta_review" in body && body.is_sample === true, renders the HTML in a sandboxed <iframe srcDoc>, decodes bundle_b64 to a Blob URL for download.
  • Error path surfaces structured {error, hint} from the route handler instead of bare "fetch failed".

#4. Real preview pipeline

  • crates/orchestrator/Cargo.toml: default = ["full"] so the binary builds against the real grokrxiv-ingest + grokrxiv-render by default.
  • routes/preview.rs switched off the stubs: bytes → grokrxiv_ingest::extract::pdf_to_text → heuristic PaperExtract (title from first non-empty line, abstract by needle-search, sections via split_sections, bibliography via extract_bibliography) → Claude → real grokrxiv-render HTML+MD+LaTeX+zip.
  • PREVIEW_MODEL env (default claude-opus-4-7). Stopped hardcoding the model.
  • Fixed the temperature is deprecated for this model error on Claude opus-4-7 by omitting the field for opus-4 / sonnet-4 model names.

#5. docker compose up --build (with Supabase preflight)

  • Switched local Postgres from a bare postgres:15-alpine container to the full Supabase CLI stack (because RLS migrations reference the anon / authenticated / service_role roles that only exist in Supabase's image).
  • docker-compose.yml now hosts only orchestrator + web; Postgres + Storage + Auth + Studio come from supabase start on the host. Web reaches Supabase via host.docker.internal:54321.
  • Orchestrator Dockerfile drops rust-toolchain.toml from the build context (clippy download was failing in CI/macos) and runs cargo build --release against the workspace lockfile directly.

#6. Real M1 persistence

  • db.rs gained upsert_paper, insert_review, insert_review_agent, set_review_artifacts helpers.
  • supervisor::run_one_paper_blocking rewritten to actually do the work: ingest via the real arXiv OAI-PMH fetcher, persist papers, loop the 6 specialist roles each making a real Claude call, persist review_agents with status / model / output / tokens / latency, render HTML+MD+TeX+ZIP to artifacts/<review_id>/, store paths on the reviews row.
  • tests/m1-pipeline.sh invokes ingest (was ingest-one). Auto-falls back to docker exec psql when host psql is missing.
  • Verified end-to-end against arxiv:2605.12484 ("Learning, Fast and Slow: Towards LLMs That Adapt Continually"): 1 papers row, 6 review_agents (one per role), reviews.status=awaiting_moderation, on-disk artifacts.

#7. Webhook + admin approve

  • Webhook payload rewritten from {branch} to {review_id, paths: ["/", "/reviews/<id>"]} to match the Next.js revalidate handler's zod schema.
  • CLI approve <review_id> is real now: reads the review row + joined paper, loads on-disk artifacts, constructs the canonical reviews/YYYY/MM/<field>/<arxiv_id>/ repo path, builds OpenReviewPr, calls publisher::open_review_pr via octocrab. Without GITHUB_TOKEN: simulated PR URL is persisted so the local-only flow stays runnable.

#8. Disclaimer policy locked

  • After the user objected to the disclaimer surfacing everywhere ("looks too stupid"), every interpolation of "Not official peer review. Not endorsed by arXiv." was removed: site footer, sample upload banner, review/paper page banners, about page subhead, all three render outputs (HTML/MD/LaTeX), PR body, seed migration's sample summary, render templates, stubs.rs.
  • PUBLIC_DISCLAIMER constants in both grokrxiv-schemas and grokrxiv-render set to empty string with a comment pointing at the single source of truth.
  • New /legal page (apps/web/app/legal/page.tsx) carries the full disclaimer with three sections (Disclaimer / Author rights / arXiv compliance).
  • Footer surfaces a quiet Legal link alongside github.com/GrokRxiv, llms.txt, API, Disputes.

#9. SEO + Google discoverability

  • CANONICAL_URL constant (apps/web/lib/env.ts) always advertises https://grokrxiv.org; SEO surfaces (sitemap, robots, root metadata, review-page metadata + JSON-LD) all point at it regardless of the dev hostname.
  • JSON-LD on /reviews/[id] is a @graph containing ScholarlyArticle (name, headline, identifier: arXiv:<id>, authors with affiliations, sameAs linking to arxiv.org/abs/, url) and Review (@id, url, name, itemReviewed.@id linked to the article, author + publisher as GrokRxiv Organization, reviewBody, datePublished, inLanguage, isAccessibleForFree).
  • Per-review metadata: title "<paper title> — GrokRxiv review of arXiv:<id>", description leads with the paper title then appends the meta_review summary tail.
  • Sitemap.xml emits absolute https://grokrxiv.org/ URLs for /, /about, /api-docs, /legal, every published review, every public paper.
  • Robots.txt declares Host: https://grokrxiv.org + Sitemap: https://grokrxiv.org/sitemap.xml + explicit Allow: / for GPTBot, ClaudeBot, Google-Extended, PerplexityBot, CCBot, Applebot-Extended.
  • Playwright spec parses the JSON-LD and asserts @graph contains both ScholarlyArticle (sameAs matches arxiv.org/abs/...) and Review (url matches grokrxiv.org/reviews/...).
