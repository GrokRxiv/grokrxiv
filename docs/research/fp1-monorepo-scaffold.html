#Architecture decisions locked here

  • Stack: Next.js 16 (Vercel) + Rust orchestrator on Railway (tokio + axum) + Supabase Postgres + GitHub GrokRxiv/reviews repo as the public artifact ledger.
  • LLM providers: Claude, Gemini, OpenAI API models, vLLM external endpoint — all behind a single LLMProvider trait.
  • Moderation: private-first. Pipeline produces private artifacts; admin approves; only then does a PR land in the public reviews repo.
  • Landing-page upload: "sample_review" path that runs a single Claude pass and returns an HTML/MD/LaTeX/zip bundle inline (no DB write).
  • GitHub org: github.com/GrokRxiv — two repos: grokrxiv (this monorepo) and reviews (canonical artifact store).
  • Ingestion scope: all 8 arXiv top-level groupings supported via grokrxiv-ingest::ALL_CATEGORIES; MVP defaults to the three (cs, math, physics-group) via DEFAULT_ACTIVE_CATEGORIES.
